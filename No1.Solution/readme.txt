чтобы можно было различным клиентам использовать сервис в различных репозиториях и с различными валидаторами
нужно ввести интерфейс для валидаторов и репозитория. А сервис будет в конструкторе принимать эти интерфейсы.
Тогда система типов будет удовлетворять принципу SRP.
Для комбинирования валидаторов можно ввести класс, который в конструкторе принимает последовательность других валидаторов.
